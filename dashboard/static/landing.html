Core capabilities

Designed for Production, Not Demos

Zero-Dependency Runtime

No Redis, RabbitMQ, or external broker required. Embedded ACID storage included.

Retries + Backoff + DLQ

Handle failure with fixed, linear, or exponential retries and inspect dead-letter jobs.

43,000+ Jobs/sec

Beats RabbitMQ on produce, consume, and end-to-end. TCP pipelining + hybrid storage for maximum throughput.

Client SDKs

Node.js, Python, and Go SDKs with zero runtime dependencies. Also embeddable as a Rust library.

Full Observability

15+ Prometheus metrics, latency histograms, per-queue gauges, pre-built Grafana dashboard, and WebSocket events.

5 Storage Backends

redb, SQLite, PostgreSQL, hybrid memory+disk, or in-memory. Swap via config.

Security Built In

Bearer token auth, rate-limited lockouts, TLS encryption, and input validation.

Cron + Interval Scheduling

Built-in schedule engine with cron expressions and interval jobs. Full CRUD with pause/resume.

DAG Workflows

Job dependencies with cycle detection, cascade failure, and flow status tracking. Build multi-step pipelines.

Webhooks

HMAC-signed HTTP callbacks on job events. Filter by queue and event type with automatic retries.

Docker Deployment

One-command with docker-compose. Includes Prometheus + Grafana monitoring stack.

Launch in minutes

From install to first job in under five minutes

# Option 1: Binary
cargo install rustqueue && rustqueue serve

# Option 2: Docker with monitoring
docker compose -f docker-compose.monitoring.yml up -d

# Push your first job
curl -X POST http://localhost:6790/api/v1/queues/emails/jobs \
  -H "Content-Type: application/json" \
  -d '{"name":"send-welcome","data":{"to":"user@example.com"}}'
