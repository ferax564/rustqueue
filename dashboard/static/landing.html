Background Jobs Without Infrastructure

Add persistent, crash-safe background job processing to any Rust application. Three lines of code. Zero operational overhead. 

let rq = RustQueue::redb("./jobs.db")?.build()?;
rq.push("emails", "send-welcome", json!({"to": "a@b.com"}), None).await?;

// That's it. The job is persisted. It survives crashes.

How it works

Start Embedded. Grow Into a Server.

1

Import the Library

Add rustqueue to your Cargo.toml. Use it directly in your app — no server process, no network, no config file.

let rq = RustQueue::redb("./jobs.db")?.build()?;
rq.push("emails", "welcome", data, None).await?;
2

Run as a Server

When you need language-agnostic access, run the same engine as a standalone server. Same data file, same guarantees.

rustqueue serve --storage ./jobs.db
3

Scale With SDKs

Connect workers in any language. Node.js, Python, and Go SDKs included — zero runtime dependencies each.

const rq = new RustQueueClient("http://localhost:6790");
await rq.push("emails", "welcome", { to: "user@a.com" });

Batteries included

Everything You Need, Nothing You Don't

Crash-Safe Storage

ACID-compliant embedded database. Jobs persist automatically. Safe to kill -9 at any time.

Retries + Backoff + DLQ

Fixed, linear, or exponential backoff. Failed jobs land in a dead-letter queue for inspection.

Cron + Interval Scheduling

Built-in schedule engine. Cron expressions and fixed intervals with pause/resume.

DAG Workflows

Job dependencies with cycle detection and cascade failure. Multi-step pipelines built in.

Webhooks + WebSocket

HMAC-signed callbacks on job events. Real-time WebSocket stream for live monitoring.

Observability

15+ Prometheus metrics, per-queue gauges, latency histograms. Pre-built Grafana dashboard.

Get started

Your First Background Job in Two Minutes

# As a library in your Rust project
cargo add rustqueue tokio serde_json anyhow

# Or as a standalone server
cargo install rustqueue && rustqueue serve

# Or with Docker
docker compose up -d
